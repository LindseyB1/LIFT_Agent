# LIFT Curated Corpus

This corpus is intentionally small and local. It gives LIFT a stable knowledge layer for resource navigation, outreach safety, and follow-up tracking. The app chunks this file by level-2 headings and cites retrieved chunks as `LIFT-CORPUS-*`.

## Resource Navigation Barriers

Resource navigation often fails because the first listed provider is not actually reachable for the user's situation. Common barriers include limited business hours, transportation requirements, location eligibility, documentation requirements, intake appointments, language access, and outdated public listings. A useful plan should identify these barriers before recommending the user spend time calling or traveling.

## Verification Before Referral

Public search results should be treated as leads, not verified referrals. Before relying on a provider, confirm the current phone number, website, hours, intake process, eligibility, service area, appointment requirements, documents required, and whether the listed service is still active. If the user needs urgent help, include a backup path that does not depend on one business-hours provider.

## Warm Outreach Principles

Warm outreach should be respectful, specific, and limited to verification unless the user has approved a direct referral. Outreach drafts should avoid unnecessary private details. A good message asks whether the provider currently offers the service, what eligibility rules apply, how intake works, and which stable contact channel should be used.

## Human Approval And Consent

LIFT should not send messages, call providers, monitor phones, scan inboxes, or access voicemail. The user remains the approval layer. The app should warn users not to enter private, classified, restricted, protected, or sensitive personal information. AI output should be reviewed before use.

## Follow-Up Tracking

A useful tracker should capture provider name, contact method, current status, next action, follow-up date, barrier identified, and outcome. Tracking repeated failures helps identify system gaps, such as transportation barriers, unavailable after-hours help, language-access gaps, or outdated public records.

## Military And Veteran Context

Military-connected users may need resources that recognize service member, veteran, caregiver, dependent, or family-readiness context. Veteran-specific programs may require documentation such as DD214 or VA enrollment. If documentation is uncertain, the plan should flag that as an eligibility question instead of assuming the user qualifies.

## Language Access

When language access is needed, the plan should ask providers directly whether interpreter services are available and whether translated forms or bilingual intake staff exist. AI-assisted translations should be treated as drafts, not official translations.
